O Processo do TDD

Primeiro se escreve o teste funcional, no próximo passo executamos o teste funcional, ele não irá passar. Sempre que teste funcional não passar, primeiro escrevemos um teste de unidade, no próximo passo executamos o teste de unidade, ele não irá passsar. Agora, sempre que teste de unidade não passsar, escrevemos um código mínimo para ele passar. Assim que o teste de unidade passa, precisamos verificar se a aplicação precisa de refatoração, se precisar, basta voltar para o passo de escrever um código mínimo. Quando temos código passando no teste de unidade, voltamos para o teste funcional, caso o teste funcional esteja passando, nós verificamos se é necessário refatorar, se for necessário, entramos novamente no ciclo de teste de unidade, se não precisar voltamos ao início do ciclo do teste funcional.
